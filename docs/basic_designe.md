# [ゲームタイトル] 基本設計書

## 1. 概要

### 1-1. 本書の目的
この資料は、要件定義書（[./gdd.md](https://github.com/path/to/docs/gdd.md)）に基づき、[ゲームタイトル]のシステム全体の構造（骨組み）と、ユーザーから見える振る舞いを定義するものです。

### 1-2. 成果物
* このドキュメント
* （もしあれば）画面遷移図（画像ファイルなど）

---

## 2. システム構成（アーキテクチャ）

このゲームを実現するための、主要な機能モジュール（パッケージや主要クラス群）の分割と、それぞれの役割を定義します。

* **Scene (シーン管理):**
    * **役割:** タイトル、メインゲーム、リザルトなどの画面（シーン）の切り替え（ライフサイクル）を管理する。
* **Input (入力管理):**
    * **役割:** キーボードやマウスからの入力を受け付け、現在のシーン（またはゲームロジック）に伝える。
* **GameLogic (ゲームロジック):**
    * **役割:** ゲームの核となるルール（例: ブロックの移動、回転、消去判定、スコア計算）を担当する。
* **Render (描画):**
    * **役割:** ゲームの状態（キャラクター、背景、UI）を画面に描画する。
* **Sound (サウンド):**
    * **役割:** BGMや効果音（SE）の再生・停止を管理する。
* **Data (データ管理):**
    * **役割:** セーブデータやハイスコアのファイルへの書き込み・読み込みを担当する。



---

## 3. 画面設計

### 3-1. 画面一覧
要件定義書で定義した画面（シーン）と、それに対応するクラス名（案）を定義します。
* タイトル画面 (例: `TitleScene.java`)
* ゲームメイン画面 (例: `GameScene.java`)
* リザルト画面 (例: `ResultScene.java`)

### 3-2. 画面遷移図
各画面がどのイベント（例: スタートボタン押下）によって、どの画面に切り替わるかを図示します。

> **[Note]**
> ここにMermaidや画像（PlantUMLなどで生成）へのリンクを貼るか、テキストベースで記述します。
> 
> (例: テキストベース)
> 1. `[START]` -> `タイトル画面`
> 2. `タイトル画面` --(スタートボタン押下)--> `ゲームメイン画面`
> 3. `ゲームメイン画面` --(ゲームオーバー)--> `リザルト画面`
> 4. `リザルト画面` --(OKボタン押下)--> `タイトル画面`

### 3-3. 各画面レイアウト（ワイヤーフレーム）
各画面に配置する主要なUI要素（ボタン、テキスト）を定義します。詳細なデザインではなく、「何がどこにあるか」の骨組みを決めます。

* **タイトル画面:**
    * [中央上部]: ゲームタイトル（画像またはテキスト）
    * [中央]: [スタート] ボタン
    * [中央下部]: [ハイスコア] 表示 (テキスト)
* **ゲームメイン画面:**
    * [中央]: プレイフィールド（描画エリア）
    * [右側]:
        * 現在のスコア（テキスト）
        * 次のブロック（描画エリア）
        * （あれば）レベル（テキスト）
* **リザルト画面:**
    * [中央]: 「GAME OVER」（テキスト）
    * [中央]: 最終スコア（テキスト）
    * [中央下部]: [タイトルに戻る] ボタン

## 4. データ設計

ゲーム内で使用するデータ構造の概要を定義します。

### 4-1. セーブデータ（永続データ）
ゲームを終了しても保持する必要があるデータ。（例: ハイスコア）

* **保存形式:** (例: JSONファイル `save.json` または プロパティファイル)
* **保存内容:**
    * `highScore`: (整数型) これまでの最高得点

### 4-2. マスターデータ
ゲームのバランスや設定を定義するデータ。（開発初期はプログラム直書き（ハードコーディング）でも良いが、将来的に分離する場合の設計）

* **ブロックデータ:**
    * 各ブロック（テトリミノなど）の形状定義
* **レベルデータ:**
    * レベルごとの敵の強さ、ブロックの落下速度など
